Hyprland config — Project notes

A compact but structure-complete cheat sheet: what gets sourced, key variables, and where keybinds/rules/gestures live.

Language:

1) Entry point and source order

The main file is hyprland.conf. It defines paths, pulls in the theme/variables, sources modular configs from hyprland/, and finally applies a user override. 

$hypr = ~/.config/hypr
$hl = $hypr/hyprland
$cConf = ~/.config/caelestia

# Theme/colors:
exec = cp -L --no-preserve=mode --update=none $hypr/scheme/default.conf $hypr/scheme/current.conf
source = $hypr/scheme/current.conf
source = $hypr/variables.conf

# User variables:
exec = mkdir -p $cConf && touch -a $cConf/hypr-vars.conf
source = $cConf/hypr-vars.conf

# Modules:
source = $hl/env.conf
source = $hl/general.conf
source = $hl/input.conf
source = $hl/misc.conf
source = $hl/animations.conf
source = $hl/decoration.conf
source = $hl/group.conf
source = $hl/execs.conf
source = $hl/rules.conf
source = $hl/gestures.conf
source = $hl/keybinds.conf

# User override:
exec = mkdir -p $cConf && touch -a $cConf/hypr-user.conf
source = $cConf/hypr-user.conf

exec-once = /usr/bin/caelestia

Tip ~/.config/caelestia/hypr-user.conf is sourced last — it’s a convenient place for local tweaks without changing this repository.

2) Theme and variables

scheme/

  • scheme/default.conf — base palette.
  • scheme/current.conf — the palette actually sourced (recreated by copying from default.conf on startup).

variables.conf

  • Single place for UX/look variables (gaps/blur/shadow/opacity/borders) and keybind “aliases”.
  • These variables are then referenced by general.conf, decoration.conf, etc.

Key variables (examples)

  • $workspaceGaps, $windowGapsIn, $windowGapsOut — gaps.
  • $windowBorderSize, $activeWindowBorderColour, $inactiveWindowBorderColour — borders.
  • $blurEnabled, $blurSize, $blurPasses — blur.
  • $shadowEnabled, $shadowRange, $shadowRenderPower — shadows.
  • $workspaceSwipeFingers, $gestureFingers, $gestureFingersMore — gestures.

3) Environment (env)

hyprland/env.conf contains environment variables for apps/toolkits (Qt/Wayland, etc.). Currently: 

env = QT_QPA_PLATFORM, wayland
env = QT_WAYLAND_DISABLE_WINDOWDECORATION, 1

4) Layout defaults (general + dwindle)

hyprland/general.conf sets the layout (dwindle), gaps/borders, and split behavior.

  • layout: dwindle
  • gaps: from variables.conf
  • dwindle: preserve_split, smart_resizing, etc.

5) Input and layouts

hyprland/input.conf configures keyboard/touchpad/cursor and small parameters.

  • kb_layout: us, ru
  • layout switching: grp:alt_shift_toggle and grp:win_space_toggle
  • touchpad: natural scroll, disable-while-typing, scroll_factor

6) Misc (behavior/background/VRR)

hyprland/misc.conf groups “misc”: VRR, logo, background, DPMS, focus.

  • VRR: misc.vrr = 1
  • background: background_color = rgb($surfaceContainer) (from the color scheme)
  • focus_on_activate: enabled

7) Animations

hyprland/animations.conf defines bezier curves and animation presets.

  • workspaces: dedicated animation for workspace switching
  • specialWorkspace: dedicated animation (slidefadevert)

8) Decoration (rounding/blur/shadow)

hyprland/decoration.conf wires rounding, blur, and shadows via variables.

  • rounding: $windowRounding
  • blur: controlled by $blurEnabled, $blurSize, $blurPasses, etc.
  • shadow: controlled by $shadowEnabled, $shadowRange, $shadowRenderPower

9) Window groups

hyprland/group.conf styles groups and the groupbar. Group navigation is typically defined in keybinds.

10) Autostart (execs)

hyprland/execs.conf starts one-time services/daemons when the session launches.

  • keyring + polkit agent
  • cliphist (clipboard history)
  • cursor setup (hyprctl setcursor + gsettings)
  • gammastep (night light)
  • caelestia shell / resizer

11) Rules

hyprland/rules.conf is the “policy layer” for windows/layers/workspaces: opacity, float, pin, special workspaces, layerrule.

Windows

  • opacity: base opacity for non-fullscreen + override
  • float: apps/dialogs that are always floating + centered
  • picture-in-picture: float + pin + keep_aspect_ratio + initial move
  • Steam: separate rules (including immediate on for games)

Special workspaces

  • windows are routed by class/initial_title to special:sysmon, special:music, special:communication, special:todo

Layer rules

  • animations/blur for launcher and shell components

12) Gestures

hyprland/gestures.conf contains workspace swipe parameters and gesture bindings.

Workspace swipe

gesture = $workspaceSwipeFingers, horizontal, workspace

Finger count is defined in variables.conf (e.g. $workspaceSwipeFingers = 3).

Gestures for special workspaces / toggles

  • there are up/down gestures to toggle todo/communication/specialws/music via caelestia toggle ...

13) Keybinds: workspaces and switching

hyprland/keybinds.conf is the main keybind list. Some combos are defined as $kb... aliases in variables.conf.

Jump to workspace by number

  • Super + 1..0 → workspace 1..10 (via wsaction.fish)
  • Ctrl + Super + 1..0 → “workspace group” (same script with -g)

Next/previous workspace

  • Super + mouse wheel → workspace -1/+1
  • Ctrl + Super + ←/→ → workspace -1/+1
  • Alt + Super + ←/→ → jump by 5 (-5/+5)
  • Super + Tab / Super + Shift + Tab → switch workspace forward/back (replaces PgUp/PgDn)

Special workspace

  • Super + S → toggle special workspace (via caelestia)

If you need “where is X configured”: most switching lives in hyprland/keybinds.conf (keys) and hyprland/gestures.conf (swipes/gestures).

14) User overrides (caelestia)

  • ~/.config/caelestia/hypr-vars.conf — extra/overridden variables.
  • ~/.config/caelestia/hypr-user.conf — any user config layered over this repo.

The idea is simple: repo files are the “base”, and these two files are the safe local layer.

This file is a snapshot-style note for the current repo state. If the config structure changes (new source, different binds/gestures), update sections 1, 12, and 13 first.

1) Точка входа и порядок подключения

Главный файл — hyprland.conf. Он задаёт пути, подтягивает тему/переменные, потом подключает модульные конфиги из hyprland/, и в конце — пользовательский оверрайд. 

$hypr = ~/.config/hypr
$hl = $hypr/hyprland
$cConf = ~/.config/caelestia

# Тема/цвета:
exec = cp -L --no-preserve=mode --update=none $hypr/scheme/default.conf $hypr/scheme/current.conf
source = $hypr/scheme/current.conf
source = $hypr/variables.conf

# Пользовательские переменные:
exec = mkdir -p $cConf && touch -a $cConf/hypr-vars.conf
source = $cConf/hypr-vars.conf

# Модули:
source = $hl/env.conf
source = $hl/general.conf
source = $hl/input.conf
source = $hl/misc.conf
source = $hl/animations.conf
source = $hl/decoration.conf
source = $hl/group.conf
source = $hl/execs.conf
source = $hl/rules.conf
source = $hl/gestures.conf
source = $hl/keybinds.conf

# Пользовательский оверрайд:
exec = mkdir -p $cConf && touch -a $cConf/hypr-user.conf
source = $cConf/hypr-user.conf

exec-once = /usr/bin/caelestia

Важно последним подключается ~/.config/caelestia/hypr-user.conf — это удобное место для локальных правок без изменения репозитория.

2) Тема и переменные

scheme/

  • scheme/default.conf — “исходная” палитра.
  • scheme/current.conf — палитра, которая реально подключается (пересоздаётся копированием из default.conf при старте).

variables.conf

  • Единое место для переменных UX/вида (gaps/blur/shadow/opacity/borders) и “ярлыков” на бинды.
  • Эти переменные затем используются в general.conf, decoration.conf, и т.д.

Ключевые переменные (примеры)

  • $workspaceGaps, $windowGapsIn, $windowGapsOut — отступы.
  • $windowBorderSize, $activeWindowBorderColour, $inactiveWindowBorderColour — рамки.
  • $blurEnabled, $blurSize, $blurPasses — blur.
  • $shadowEnabled, $shadowRange, $shadowRenderPower — тени.
  • $workspaceSwipeFingers, $gestureFingers, $gestureFingersMore — жесты.

3) Окружение (env)

hyprland/env.conf — переменные окружения для приложений/тулкитов (Qt/Wayland и др.). Сейчас явно выставлено: 

env = QT_QPA_PLATFORM, wayland
env = QT_WAYLAND_DISABLE_WINDOWDECORATION, 1

4) Общие настройки лейаута (general + dwindle)

hyprland/general.conf задаёт layout (dwindle), gaps/borders и поведение разбиений.

  • layout: dwindle
  • gaps: берутся из variables.conf
  • dwindle: preserve_split, smart_resizing и т.д.

5) Ввод (input) и раскладки

hyprland/input.conf — клавиатура/тачпад/курсор и мелкие параметры.

  • kb_layout: us, ru
  • переключение раскладки: grp:alt_shift_toggle и grp:win_space_toggle
  • touchpad: natural scroll, disable-while-typing, scroll_factor

6) Misc (поведение/фон/VRR)

hyprland/misc.conf — “разное”: VRR, лого, фон, DPMS, фокус.

  • VRR: misc.vrr = 1
  • фон: background_color = rgb($surfaceContainer) (цвет из схемы)
  • focus_on_activate: включён

7) Анимации

hyprland/animations.conf — кривые (bezier) и наборы анимаций.

  • workspaces: отдельная анимация для переключения рабочих столов
  • specialWorkspace: отдельная анимация (slidefadevert)

8) Декор (скругления/blur/shadow)

hyprland/decoration.conf подключает скругления, blur и тени через переменные.

  • rounding: $windowRounding
  • blur: управляется $blurEnabled, $blurSize, $blurPasses и др.
  • shadow: управляется $shadowEnabled, $shadowRange, $shadowRenderPower

9) Группы окон (group)

hyprland/group.conf — оформление групп и groupbar. Переключение активного окна в группе обычно настраивается биндами.

10) Авто‑запуск (execs)

hyprland/execs.conf — сервисы/демоны, которые стартуют один раз при запуске сессии.

  • keyring + polkit agent
  • cliphist (история буфера обмена)
  • курсоры (через hyprctl setcursor и gsettings)
  • gammastep (ночной свет)
  • caelestia shell / resizer

11) Правила (rules)

hyprland/rules.conf — “политика” для окон/слоёв/воркспейсов: прозрачность, float, pin, special workspace, layerrule.

Окна

  • прозрачность: базовая opacity для не‑fullscreen + override
  • float: набор приложений/диалогов, которые всегда плавающие + центрирование
  • picture‑in‑picture: float + pin + keep_aspect_ratio + стартовый move
  • Steam: отдельные правила (в т.ч. immediate on для игр)

Special workspaces

  • окна по class/initial_title автоматически улетают в special:sysmon, special:music, special:communication, special:todo

Layer rules

  • анимации/blur для launcher и компонентов shell

12) Жесты (gestures)

hyprland/gestures.conf — параметры свайпа рабочих столов и сами жесты.

Свайп рабочих столов

gesture = $workspaceSwipeFingers, horizontal, workspace

Количество пальцев берётся из variables.conf (например $workspaceSwipeFingers = 3).

Жесты для special workspaces / toggles

  • есть жесты up/down для переключения todo/communication/specialws/music через caelestia toggle ...

13) Клавиши (keybinds): рабочие столы и переключение

hyprland/keybinds.conf — главный набор биндoв. Часть модификаторов/комбо вынесена в variables.conf как $kb....

Переход на workspace по номеру

  • Super + 1..0 → workspace 1..10 (через wsaction.fish)
  • Ctrl + Super + 1..0 → “workspace group” (тот же скрипт с флагом -g)

Следующий/предыдущий workspace

  • Super + колесо мыши → workspace -1/+1
  • Ctrl + Super + ←/→ → workspace -1/+1
  • Alt + Super + ←/→ → прыжок на 5 воркспейсов (-5/+5)
  • Super + Tab / Super + Shift + Tab → переключение workspace вперёд/назад (замена PgUp/PgDn)

Special workspace

  • Super + S → toggle special workspace (через caelestia)

Если нужно быстро найти “что отвечает за X”: почти всё, что связано с переключениями, находится в hyprland/keybinds.conf (клавиши) и hyprland/gestures.conf (свайпы/жесты).

14) Пользовательские оверрайды (caelestia)

  • ~/.config/caelestia/hypr-vars.conf — дополнительные/перезаписанные переменные.
  • ~/.config/caelestia/hypr-user.conf — любые пользовательские настройки поверх репозитория.

Логика простая: всё из репозитория — “база”, а эти два файла — безопасное место для локальных правок.

Этот файл — конспект текущего состояния репозитория. Если структура конфигов меняется (новые source, другие бинды/жесты), обнови разделы 1, 12 и 13 в первую очередь.

source = $cConf/hypr-user.conf

exec-once = /usr/bin/caelestia

نکته ~/.config/caelestia/hypr-user.conf آخر از همه source می‌شود؛ جای مناسبی برای تغییرات محلی بدون دست‌زدن به این ریپو است.

۲) تم و متغیرها

scheme/

  • scheme/default.conf — پالت پایه.
  • scheme/current.conf — پالتی که واقعاً source می‌شود (در شروع با کپی از default.conf بازتولید می‌شود).

variables.conf

  • محل واحد برای متغیرهای ظاهر/UX (gaps/blur/shadow/opacity/borders) و aliasهای keybind.
  • این متغیرها در general.conf، decoration.conf و… استفاده می‌شوند.

نمونه متغیرهای مهم

  • $workspaceGaps, $windowGapsIn, $windowGapsOut — فاصله‌ها.
  • $windowBorderSize, $activeWindowBorderColour, $inactiveWindowBorderColour — مرز پنجره.
  • $blurEnabled, $blurSize, $blurPasses — محوشدگی.
  • $shadowEnabled, $shadowRange, $shadowRenderPower — سایه.
  • $workspaceSwipeFingers, $gestureFingers, $gestureFingersMore — ژست‌ها.

۳) محیط (env)

hyprland/env.conf شامل متغیرهای محیطی برای اپ‌ها/Toolkitهاست (Qt/Wayland و…). 

env = QT_QPA_PLATFORM, wayland
env = QT_WAYLAND_DISABLE_WINDOWDECORATION, 1

۴) پیش‌فرض‌های چیدمان (general + dwindle)

hyprland/general.conf layout (dwindle)، gaps/borders و رفتار split را تنظیم می‌کند.

  • layout: dwindle
  • gaps: از variables.conf
  • dwindle: preserve_split, smart_resizing و…

۵) ورودی (input) و Layoutها

hyprland/input.conf کیبورد/تاچ‌پد/کرسر و پارامترهای ریز را تنظیم می‌کند.

  • kb_layout: us, ru
  • تعویض زبان: grp:alt_shift_toggle و grp:win_space_toggle
  • touchpad: natural scroll, disable-while-typing, scroll_factor

۶) Misc (رفتار/پس‌زمینه/VRR)

hyprland/misc.conf تنظیمات متفرقه مثل VRR، پس‌زمینه و focus را نگه می‌دارد.

  • VRR: misc.vrr = 1
  • پس‌زمینه: background_color = rgb($surfaceContainer)
  • focus_on_activate: روشن

۷) انیمیشن‌ها

hyprland/animations.conf منحنی‌ها و presetهای انیمیشن را تعریف می‌کند.

  • workspaces: انیمیشن مخصوص جابه‌جایی workspace
  • specialWorkspace: انیمیشن مخصوص (slidefadevert)

۸) دکور (گوشه‌گرد/blur/shadow)

hyprland/decoration.conf گوشه‌گردی، blur و سایه را با متغیرها کنترل می‌کند.

  • rounding: $windowRounding
  • blur: با $blurEnabled، $blurSize، $blurPasses و…
  • shadow: با $shadowEnabled، $shadowRange، $shadowRenderPower

۹) گروه پنجره‌ها

hyprland/group.conf ظاهر group و groupbar را تنظیم می‌کند؛ جابه‌جایی معمولاً در keybinds است.

۱۰) اجرای خودکار (execs)

hyprland/execs.conf سرویس‌هایی که یک‌بار هنگام شروع سشن اجرا می‌شوند را راه‌اندازی می‌کند.

  • keyring + polkit agent
  • cliphist (تاریخچه کلیپ‌بورد)
  • تنظیم کرسر (hyprctl setcursor + gsettings)
  • gammastep (نور شب)
  • caelestia shell / resizer

۱۱) قوانین (rules)

hyprland/rules.conf “policy” برای پنجره/لایه/workspace است: opacity, float, pin, special workspace, layerrule.

پنجره‌ها

  • opacity: شفافیت پایه برای غیر تمام‌صفحه + override
  • float: اپ‌ها/دیالوگ‌های شناور + center
  • PiP: float + pin + keep_aspect_ratio + move اولیه
  • Steam: قوانین جدا (از جمله immediate on برای بازی‌ها)

Special workspaces

  • مسیر‌دهی پنجره‌ها به special:sysmon, special:music, special:communication, special:todo

Layer rules

  • انیمیشن/blur برای launcher و اجزای shell

۱۲) ژست‌ها (gestures)

hyprland/gestures.conf پارامترهای swipe workspace و binding ژست‌ها را نگه می‌دارد.

Swipe بین workspaceها

gesture = $workspaceSwipeFingers, horizontal, workspace

تعداد انگشت از variables.conf می‌آید (مثلاً $workspaceSwipeFingers = 3).

ژست‌های toggle برای special workspaceها

  • ژست‌های up/down برای todo/communication/specialws/music از طریق caelestia toggle ...

۱۳) کلیدها: workspace و جابه‌جایی

hyprland/keybinds.conf لیست اصلی keybindهاست؛ بعضی ترکیب‌ها در variables.conf با $kb... نام‌گذاری شده‌اند.

رفتن به workspace با شماره

  • Super + 1..0 → workspace 1..10 (با wsaction.fish)
  • Ctrl + Super + 1..0 → workspace group (همان اسکریپت با -g)

workspace بعدی/قبلی

  • Super + wheel → workspace -1/+1
  • Ctrl + Super + ←/→ → workspace -1/+1
  • Alt + Super + ←/→ → پرش ۵تایی (-5/+5)
  • Super + Tab / Super + Shift + Tab → جلو/عقب (جایگزین PgUp/PgDn)

Special workspace

  • Super + S → toggle special workspace (با caelestia)

برای پیدا کردن “X کجاست”: بیشتر سوییچ‌ها در hyprland/keybinds.conf و hyprland/gestures.conf هستند.

۱۴) Overrideهای کاربر (caelestia)

  • ~/.config/caelestia/hypr-vars.conf — متغیرهای اضافه/override.
  • ~/.config/caelestia/hypr-user.conf — تنظیمات کاربر روی این ریپو.

ایده ساده است: فایل‌های ریپو پایه‌اند و این دو فایل لایهٔ محلی امن هستند.

این فایل یک “یادداشت از وضعیت فعلی ریپو” است. اگر ساختار کانفیگ عوض شد (sourceهای جدید یا bind/gestureهای متفاوت)، اول بخش‌های ۱، ۱۲ و ۱۳ را به‌روزرسانی کن.
